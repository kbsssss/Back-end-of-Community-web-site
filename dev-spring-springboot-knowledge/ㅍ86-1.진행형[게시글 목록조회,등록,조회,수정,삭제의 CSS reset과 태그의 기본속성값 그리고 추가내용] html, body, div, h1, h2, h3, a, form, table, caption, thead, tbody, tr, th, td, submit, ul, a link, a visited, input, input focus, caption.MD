    (1).<html>
        (1-1).<html> 영역
            이 <html>태그는 블록요소인데, 그 안의 <body>태그도 블록요소이다. 근데 아래보면 알겠지만, 부모자식요소가 둘다 블록요소이고 자식
            요소의 위와 아래는 border까지만 부모요소의 contents영역으로 본다. 근데, <html>태그는 희안하게 예외적으로 하위요소들의 마진 부분까지도
            컨텐츠영역이라고 나와있다.(실제로 <h2>는 상하 마진이 있는데 <body>에서는 이 마진 부분을 컨텐츠영역으로 포함시키지 않았다. 근데 <html>는 포함시킴 그것도 상,하 전부, 하는 <body>맨 아래에 <h1>써서 확인)
            또한, 실제로 해본게 <div>중복으로 하위요소들을 겹겹히 하고 제일 하위에 <h2>를했는데도, 부모요소뿐만 아니라 상위요소들의 블록요소는 하위요소의 상,하 마진영역을 컨텐츠영역으로
            포함시키지 않았다. 
            코드 ex)
            <!DOCTYPE html>
            <html lang="ko">
            <head>
                <title>세번째</title>
            </head>
            <body>
            <div>
                <h1>b</h1>
            </div>
            </body>
            </html>
            [참조문헌 : HTML5책 315pg] / 여기보면, 마진병합이 이루어지는경우 3가지(부모자식요소, 형제요소, 한요소)와 왜 상하만 마진병합이 이루어지고 오른쪽왼쪽은 마진병합이 이루어지지않는지 inline요소도 나와있다.   
            [참조문헌 : 제이쿼리 287pg] / 여기에 부모 자식요소가 마진병합이 어떻게 이루어지는지, 부모자식이 둘다 블록요소일때 자식요소의 상하는 border까지만 부모요소에서 컨텐츠영역으로 여기고, margin은 아예 안따진다. 또한 왼,오른쪽은 마진까지 부모요소의 contents영역으로 본다.
            [직접해봄]
        (1-2).<html>의 font-size는 애초에 브라우저에서 16px을 갖고오는거로 보고 있다고 한다.
            아래 참조문헌에는 그러한데, 이게 html의 font-size를 200%로 하면 그 아래 요소들도 font-size가 32px이 된다.
            즉, <html>요소 자체가 font-size가 16px이라기 보다 브라우저에서 16px 받아온다라고 이해하는게 나을것 같다.
            그리고 두번째 참조문헌 보면, <body>요소의 기본 font-size가 16px인데 그러면, <html>의 원래 기본 font-size도
            16px인것을 알 수 있다.
